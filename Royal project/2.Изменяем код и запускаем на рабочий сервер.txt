Для начала создадим jar файл проекта

Предварительные настройки для этого в проекте уже выполнены и находятся в файле pom.xml
Итоговый файл получится royal.jar
Файл будет расположен в подкаталоге /target проекта. Путь к проекту виден в заголовке структуры проекта, по правому клику мышки его оттуда можно скопировать

Чтобы собрать файл вызываем из меню Maven шаг package из Lifecycle
Для очистки /target можно использовать шаг clean

Следующим шагом проверим работоспособность созданного jar файла
Выполняем команду в консоли cmd
java -jar <полное имя файла jar>

В случае если java не является опознаной командой системы, нужно прописать java в PATH переменные среды (как сделать гуглим рецепт под свою операционку)

При начале выполнения кода в консоли должен быть выведен логотип Spring и данные о запущенном сервере. После чего можно проверять факт запуска приложения из браузера
Завершение выполнения кода в консоли Ctrl+C

На этом шаге у нас есть jar файл, переместим его на рабочий сервер

Воспользуемся утилитой pscp из комлекта PuTTy, ранее установленного. Сама утилита PuTTy нам тоже понадобится далее
Выполняем команду в консоли cmd
pscp D:\royal.jar root@185.231.245.49:/usr/share/royal
Где D:\royal.jar путь к файлу jar созданному ранее.

Теперь наш файл на сервере обновлен
Можем запустить его на сервере командой
java -jar /usr/share/royal/royal.jar
и проверить
http://185.231.245.49:8888/greeting?name=Novice
